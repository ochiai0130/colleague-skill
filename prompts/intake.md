# 基本情報入力スクリプト

## オープニング

```
この同僚の Skill を作成します。3つの質問に答えるだけで、どれもスキップ可能です。
```

---

## 質問シーケンス

### Q1：ニックネーム/コードネーム

```
この同僚は何と呼ばれていますか？（ニックネーム、あだ名、コードネームなど、複数語は - で接続）

例：qing-yun
```

- 任意の文字列を受け付ける
- 生成される slug は統一して `-` で接続（アンダースコアは使用しない）
- 中国語は自動的にピンインに変換して `-` で接続（"青云" → `qing-yun`、"小李" → `xiao-li`）
- 英語はそのまま小文字にして `-` で接続（"Big Mike" → `big-mike`）

### Q2：基本情報

会社、職級、職種、性別を1つの質問にまとめ、ユーザーに一言で回答してもらう：

```
一言で基本情報を教えてください——会社、職級、職種、性別、思いつくものを何でも。スキップも可。

例：字節 2-1 バックエンドエンジニア 男性
```

ユーザーの回答から以下のフィールドを解析する（不足分は空欄）：
- **会社**
- **職級**
- **職種**
- **性別**

#### 職級対照参考表

| 会社 | 職級形式 | エンジニア/研究員 | シニアエンジニア | エキスパート/スペシャリスト | Staff/Principal |
|------|---------|------------|---------|---------|----------------|
| 字節跳動 | X-Y | 2-1, 2-2 | 3-1, 3-2 | 3-3 | 3-3+（O級） |
| 阿里巴巴 | P級 | P5, P6 | P7 | P8 | P9+ |

**企業間のおおよその対応関係**：

```
字節 2-1/2-2  ≈  阿里 P6   ≈  腾讯 T2  ≈  百度 T6
字節 3-1      ≈  阿里 P7   ≈  腾讯 T3-1 ≈  百度 T7

```

> 注：字節 2-1 はエンジニア職、3-1 からシニアエンジニア；
> 2-1 は阿里 P6 に相当し、独立してタスクを遂行できる主力エンジニアレベル。

### Q3：性格プロフィール

MBTI、星座、個性タグ、企業文化タグ、主観的印象をすべてまとめて、ユーザーに自由に記述してもらう：

```
一言で性格を教えてください——MBTI、星座、個性の特徴、企業文化の影響、あなたの印象、
思いつくものを何でも。スキップも可。

例：INTJ やぎ座 責任転嫁の達人 字節スタイル CRは厳しいが理由を説明しない
```

ユーザーの回答から以下のフィールドを識別・抽出する（不足分は空欄）：
- **MBTI**：16種の標準タイプ
- **星座**：12星座
- **個性タグ**：下記タグライブラリからマッチング、カスタム記述も受け付ける
- **企業文化タグ**：下記タグライブラリからマッチング
- **主観的印象**：分類できない自由記述、原文をそのまま保持

#### 個性タグライブラリ

**仕事への姿勢**：真面目で責任感が強い / まあまあでいい / 責任転嫁の達人 / 責任を被る人 / 完璧主義 / 先延ばし癖

**コミュニケーションスタイル**：ストレート / 遠回し / 寡黙 / おしゃべり / ボイスメッセージ好き / 既読スルー / 既読で的外れな返信 / 即レス強迫症

**意思決定スタイル**：即断即決 / 二転三転する / 上司頼み / 強引に推進 / データドリブン / 完全に直感

**感情スタイル**：感情が安定している / 繊細すぎる / 興奮しやすい / 冷淡で距離がある / 表面上は穏やか / 皮肉っぽい

**話術と手段**：PUA の達人 / 社内政治プレイヤー / 責任転嫁アーティスト / 上司マネジメントの専門家 / 説教好き / 感情的な脅し

#### 企業文化タグライブラリ

- **字節スタイル** — 率直で直接的、impact を追求、必ず context から話す、「すり合わせ」が口癖
- **阿里スタイル** — 六脈神剣、「エンパワメント」「グリップ」「エコシステム」「クローズドループ」を多用
- **腾讯スタイル** — データで語る、馬レース方式、慎重で保守的、ユーザー体験重視
- **華為スタイル** — 奮闘者文化、プロセスと規範重視、PPT報告を好む、実行力を強調
- **百度スタイル** — 技術至上主義、階層意識が強い、社内競争が激しい
- **美団スタイル** — 極限の実行力、細部にこだわる、ローカル思考
- **第一原理** — マスク式、本質を追求、類推を拒否、ラディカルな簡素化
- **OKR 信奉者** — 何事もまず Objective を問う、KR にこだわる
- **大企業ライン** — 規範は整っているが創造力が低い、SOP に依存、責任を負うのを恐れる
- **スタートアップ派** — リソースが限られている、フルスタック思考、結果重視、混乱を許容する
