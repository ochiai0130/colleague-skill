# 差分マージプロンプト

## タスク

以下を受け取ります：
1. 既存の `work.md` の内容
2. 既存の `persona.md` の内容
3. 新しい原材料の内容（ファイルまたはメッセージ）

あなたのタスクは、新しい内容がどの部分を更新すべきかを判断し、差分更新の内容を出力することです。

**原則：差分のみ追加し、既存の結論は上書きしない。矛盾がある場合は、矛盾の通知を出力してユーザーに判断を委ねる。**

---

## Step 1：分類判定

新しい内容の各情報を分類する：

| 情報タイプ | 分類先 |
|---------|------|
| 技術規範、コードスタイル、API設計、業務フロー | → work.md |
| 業務知識、システム職責、技術的結論 | → work.md |
| コミュニケーションスタイル、口癖、表現の習慣 | → persona.md |
| 意思決定行動、対人関係、感情パターン | → persona.md |
| 両方に該当 | → それぞれに分類 |

---

## Step 2：矛盾チェック

新しい内容と既存の内容を比較する：

- 新しい内容が既存の情報を**補完**する場合（新しい詳細が追加された）→ そのまま追加
- 新しい内容が既存の情報を**確認**する場合 → 無視（重複記載しない）
- 新しい内容が**既存の情報と矛盾**する場合 → 矛盾通知を出力：

```
⚠️ 矛盾を検出：
- 既存：{既存の記述}
- 新発見：{新しい内容の記述}
- 出典：{ファイル名/日時}

提案：[既存を維持 / 新しい内容に更新 / 両方を保持し日時を注記]
ユーザーの判断をお願いします。
```

---

## Step 3：更新パッチの生成

`work.md` への更新、出力形式：
```
=== work.md 更新 ===

[「技術規範/命名規則」セクションに追加]
- {新しい内容}

[「経験ナレッジベース」セクションに追加]
- {新しい知識結論}

[更新なし] または [上記セクションに更新あり]
```

`persona.md` への更新、出力形式：
```
=== persona.md 更新 ===

[「Layer 2/語彙の習慣」セクションに追加]
- 新しい口癖："{xxx}"

[「Layer 4/同僚に対して」セクションに追加]
- {新しい行動の記述}

[更新なし] または [上記セクションに更新あり]
```

---

## Step 4：更新サマリーの生成

ユーザーに表示する：
```
今回の更新サマリー：
- work.md：{N} 件の新しい情報を追加（{簡単な説明}）
- persona.md：{N} 件の新しい情報を追加（{簡単な説明}）
- {N} 件の矛盾を検出、確認が必要です（上記を参照）

バージョンは {vN} から {vN+1} にアップグレードされます。
更新を適用しますか？
```
