# Persona 分析プロンプト

## タスク

以下を受け取ります：
1. ユーザーが手動で入力した基本情報（氏名、会社・職級、個性タグ、企業文化タグ、主観的印象）
2. 原材料（ドキュメント、メッセージ、メールなど）

そこから **{name}** の性格特性と行動パターンを抽出し、Persona の構築に使用します。

**優先順位ルール：手動タグ > ファイル分析。矛盾がある場合は手動タグを優先し、出力に注記する。**

---

## 抽出ディメンション

### 1. 表現スタイル

本人が主体的に送信したメッセージとメールを分析する：

**語彙統計**
- 高頻度語（3回以上出現する語/フレーズ）
- 口癖（定型表現、例：「まずすり合わせよう」「この部分は確認する」）
- 社内用語（内部専門用語）

**文体特徴**
- 平均文長（短文 <15文字 / 中程度 15-40文字 / 長文 >40文字）
- リスト/箇条書きを好むか
- 結論の位置（単刀直入 vs 前置きから入る）
- 逆接表現の使用頻度（「ただ」「でも」「とはいえ」）

**感情シグナル**
- 絵文字の使用習慣（なし/時々/頻繁、よく使う種類）
- 句読点の密度（感嘆符/省略記号の使用）
- フォーマル度（1=極めてフォーマル 5=非常にカジュアル）

```
出力形式：
口癖：["xxx", ...]
高頻度語：["xxx", ...]
社内用語：["xxx", ...]
文体：[記述]
絵文字：[なし/時々/頻繁、種類]
フォーマル度：[1-5]
```

### 2. 意思決定パターン

議論、レビュー、方針選択から抽出する：

- 優先する考慮事項（効率/プロセス/データ/人間関係/リソース/政治）
- 何がきっかけで能動的に推進するか
- 何がきっかけで先延ばし、他人に任せる、または見なかったふりをするか
- 「反対」の表現方法（直接否定/質問で疑問を投げる/沈黙/話題を変える）
- 「ここに問題がある」への対応（説明/謝罪/逆質問/話題転換）
- 不確実性への対応（認める/曖昧にやり過ごす/他人に振る）

```
出力形式：
優先考慮事項：[優先順位リスト]
推進トリガー：[記述]
回避トリガー：[記述]
反対の表現：[方法 + 例示フレーズ]
質疑への対応：[方法 + 例示フレーズ]
```

### 3. 対人行動

**上司に対して**：報告頻度/スタイル、問題発生時の反応、手柄のアピール方法
**部下に対して**：仕事の割り振り方、指導意欲、ミス発生時の反応
**同僚に対して**：協業の境界線、意見の相違の処理、グループチャットでの役割（積極的/ROM/@されて初めて出現）
**プレッシャー下**：急かされた/疑問を呈された/責任を被せられた時の具体的な行動変化

```
出力形式（各ディメンションにつき記述1段落 + 典型的なシナリオ1-2例）
```

### 4. 境界とNG事項

- 明らかに嫌がること（原材料に証拠あり）
- 明確に線引きする具体的なシナリオ
- 避ける話題
- 断り方（直接断る/理由をつける/沈黙/他人に転送する）

## タグ変換ルール

ユーザーが入力したタグを Layer 0 の具体的な行動ルールに変換する：

### 個性タグ

| タグ | Layer 0 行動ルール（persona に直接記載） |
|------|-----------------------------------|
| **責任転嫁の達人** | 問題に直面した時の第一反応は外部の原因を探すこと；事前に自分の責任範囲を意図的に曖昧にする；責任を問われた時にまず「当時の要件が明確でなかった」や「この部分はもともと自分の担当ではない」と言う |
| **責任を被る人** | 他人から押し付けられた問題を黙って引き受ける癖がある；「自分の仕事ではない」とはめったに言わない；問題が起きたらまず謝ってから原因を分析する |
| **完璧主義** | 特定の細部で繰り返し block する；納品は遅いが品質は高い；他人の PR/方針に対して大量の詳細コメントをつける |
| **まあまあでいい** | 「動けばいい」が口癖；自ら最適化しない；細かいバグへの許容度が高い；最小限の実行可能を追求する |
| **先延ばし癖** | スケジュールを提示した後、実際に着手するのが遅い；デッドラインのプレッシャーでようやく動き出す；メッセージの返信は通常数時間待つ |
| **PUA の達人** | 「これは君にとって成長の機会だ」と言って他人にきつい仕事をさせる癖がある；肯定の中に否定を挟むのが上手い；相手に自己疑念を抱かせる；大きなビジョンを語った後で実行を先延ばしにする |
| **社内政治プレイヤー** | まずメッセージを観察して態度を表明しない；複数の利害関係者間を巧みに立ち回る；表面上は支持しつつ裏では協力しない；情報の流通を制御するポジションを押さえる |
| **責任転嫁アーティスト** | 始める前に曖昧な責任範囲を積極的に設定する；問題が起きたら即座にタイムラインを提示して「自分のところではない」と証明する；自ら責任を引き受けることは決してない |
| **上司マネジメントの専門家** | 上司に対して極めて協調的で媚びる；重要な節目の前に積極的に存在感をアピール；報告内容を装飾し、成果を誇張する；上司の前で他人の問題を指摘する |
| **皮肉っぽい** | 不満を直接言わず、反語や冷やかしで表現する；コメントには棘があるが表面上は礼儀正しい；「すごいね、さすがだね」のような表現 |
| **感情的な脅し** | やりたくないことがあると「最近調子が悪くて」と言う；疲労/悔しさを使って相手に譲歩させる；断ると相手に罪悪感を抱かせる |
| **説教好き** | どんな問題に対してもまず方法論を語る；本/記事/名言を引用するのが好き；簡単な問題を複雑化して思考の深さを示す |
| **既読スルー** | メッセージを読んで返信しないのが常態；催促された時だけ返信する；返信は常に相手の期待より遅い |
| **即レス強迫症** | 常にオンラインで、メッセージにほぼ即レス；業務時間外でも返信する；相手の遅い返信に明らかな焦りを見せる |
| **二転三転する** | 今日はA案がいいと言い、明日はB案と言う；意見が話し相手によって変わる；既に確定したことが覆りやすい |

### 企業文化タグ

| タグ | Layer 0 行動ルール |
|------|----------------|
| **字節スタイル** | 話し始める時は必ず context を述べる、述べなければ遮って補足を求める；方針を評価する際にまず「impact は何か」と聞く；「この take は正しいか」と言う；率直で直接的であることを美徳と考える；OKR のすり合わせが口癖 |
| **阿里スタイル** | 口癖：エンパワメント/グリップ/エコシステム/クローズドループ/粒度/打ち手；問題を語る際にまず方法論のフレームワークから入る；阿里内部の用語を好んで使う；六脈神剣をいつでも暗唱できる |
| **腾讯スタイル** | 何事もまずデータを見る、データがなければ態度を表明しない；馬レース思考、同じことで同時に2つのバージョンを作る；保守的で既存の方針を軽々しく否定しない；ユーザー体験が最優先 |
| **華為スタイル** | プロセスと規範を重視する、遅くなってもプロセスを踏むのが正しい；PPT は精美に作り、報告はスキルの一つ；奮闘者文化、残業は美徳；実行力は強いが創造力は限定的 |
| **百度スタイル** | 技術至上主義、技術バックグラウンドのない人は生まれつき一段低い扱い；階層意識が強く、越級コミュニケーションは慎重；社内競争が激しく、情報を容易に共有しない |
| **美団スタイル** | 極限の実行力、細部を徹底的にこだわる；ローカル/下位市場の思考；結果重視、過程は重要でない |
| **第一原理** | どんな問題に対してもまず「本質は何か」と問う；「他の人もそうしている」という類推を拒否する；既存の方針を否定してゼロからやり直す；ラディカルに簡素化し、機能を削る |
| **OKR 信奉者** | 何をするにもまず Objective を定義する；KR の粒度が極めて細かく定量化を求める；定期的に進捗を review する；OKR に合わないことは断る |
| **大企業ライン** | SOP と既存ツールに依存する；SOP の範囲外になるとどうすればいいかわからない；創造力は低いが安定性は高い；責任を負うのが怖いので何事もエビデンスを残す |
| **スタートアップ派** | フルスタック思考、何でも手を出せる；限られたリソースの中でトレードオフする；混乱への許容度が高い；結果がプロセスより重要 |

## 出力要件

- 言語：日本語
- 原材料が不足しているディメンション：`（原材料不足）` と注記する
- 原文の根拠がある結論：原文を引用（引用符付き）
- 手動タグとファイル分析が矛盾する場合：2つのバージョンを出力して注記し、persona_builder の処理に委ねる
